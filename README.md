The objective of my project is to develop a native Android application in Kotlin using Android Studio that enables users to study and validate life hacks through flashcards. The core objective of the app is to provide a seamless interface for users to test their understanding of lifestyle hacks and common myths. It aims to debunk misconceptions and reinforce information through active engagement. The name of the app is Lifestyle Hacks or Myths.
Users are presented with a series of statements and must determine their validity using ‘Hack” or “Myth’ inputs. For every response, the app provides a detailed explanation. This ensures the users learns the reasoning behind the correct answer, rather than just the answer itself. The system monitors the user’s movement through the question set, maintaining an accurate tally of the total score. Upon completion of the quiz, the app transitions to a dedicated summary screen, providing the user with their results.
The main screen consists of multiple text view elements designed with distinct hierarchy. Displays the app title which is Lifestyle Hack or Myth styled in italic. Secondary text, displays a descriptive tagline called “Smart Hacks, Real Facts ,No Myths ‘in a bold text style to provide contrast against the main heading The Start  button element is the primary call to action for the user .This layout effectively guides the users eyes from title at the top ,through the descriptive text in the middle, down  to the interactive button at the bottom. The quiz screen includes two text views for displaying questions and feedback, along with three buttons labelled ‘Myth,’ ‘Hack,’ and ‘Next.’ The score screen features two text views one displaying the user’s score and another providing an encouragement message along with a ‘Review’ button.
The logic of the application, the core of the quiz is a list of custom objects. Each objects contains a String for the question text, a Boolean for the correct answer, a String for the explanation or feedback. When user clicks True or False button, the value of the button pressed is compared against the answer property of the object at the current list index. If the comparison is true, a score variable is incremented. A message is generated using the explanation string from the current object to inform the user why they were right or wrong. After the feedback is provided, the app must decide whether to continue or end. This is managed by comparing the current Index to the total size of the list questions. If more questions remain increment currentIndex by 1. Quiz completes the app triggers Intent
The app is bult in Android Studio, leveraging modern development practices:
•	Language: Kotlin 
•	Logic: Implementation of list-based questions handling and state management to track scores
•	Navigation: Intent based transitions to move users from the active quiz environment to results display 
The application guides users through introduction and navigation on the main screen, supports active participation on the quiz screen, and concludes with results analysis and review options on the score screen.”
